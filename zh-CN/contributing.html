贡献指南¶

感谢你有兴趣为 sphinx-book-theme 做出贡献! 你真了不起✨。

本项目遵循 Executable Books Project 贡献指南。它包含了关于我们在编码风格、拉取请求工作流程、提交信息等方面的惯例。

这个页面包含的信息可以帮助你开始这个项目的开发。

储存库的结构¶

这个资源库被分割成几个关键的文件夹：

sphinx-book-theme/

Sphinx 插件包，包含 Python 代码、Jinja 模板和（编译的）assets（HTML/CSS/JS 等）。

只要网站是用 Sphinx 建立的，这些都是用来为你的网站中的每个文件生成 HTML 页面。

注意：不要直接改变已编译的 CSS/JS（在 src/ 中改变）。

src/

包含源码 SCSS 和 JS，将被编译并写入 sphinx-book-theme/static，由 web-compile-config.yml 配置（详见 executablebooks/web-compile）。

默认情况下，在开发命令期间，该编译被调用（见下文）。

docs/

主题的文档，旨在包括所有核心的 Sphinx “组件”（列表、训诫等），以检查/显示它们在这个主题中是如何表现的。

包含在 conf.py中的 build 配置。

docs/reference

文档的参考部分包含了主题的外观和感觉的参考材料。“kitchen sink”是直接从 sphinx-themes 网站提取的。还有其他部分是关于主题的特定元素”

tests/

测试基础设施，使用 pytest 和 beautifulsoup 来验证生成的 HTML 是否为我们所期望的。

这些测试大多也使用 pytest-regressions，以检查新生成的 HTML 是否与以前生成的 HTML 不同。

.github/workflows/

包含持续集成（CI）工作流程，在提交/PR 到 GitHub 仓库时运行。

设置开发环境¶

  1. 使用 git 获取这个项目的源代码：

    git clone https://github.com/executablebooks/sphinx-book-theme
cd sphinx-book-theme

  2. 确保你有 Python 3.6 或更新的版本！

  3. 安装 tox。tox 是一个工具，用于管理测试套件的虚拟环境或与版本库一起运行的普通作业。它可以确保你的环境在每次构建文档时都是一致的。

    $ pip install tox

    你可以这样列出所有的 tox 环境。

    $ tox -a

    每一个都对应着一个测试环境或一个建立文档的任务，并有自己的虚拟环境”

    参见 sphinx-book-theme/tox.ini，了解所有可用的开发环境的细节。

  4. 安装pre-commit。我们使用 pre-commit 来确保任何时候提交的代码都符合某些标准。首先，按照 pre-commit 的安装说明。然后，在与版本库相同的文件夹中运行以下命令：

    $ pre-commit install

    注解

    你也可以通过 tox 运行 pre-commit。

    $ tox -e py38-pre-commit -- --all

    或手动运行该版本库的所有 pre-commit 工作。

    $ pre-commit run --all-files

预览你对主题所做的改动¶

预览你对这个主题所做的修改的最简单的方法是通过使用 tox 在本地建立这个主题的文档。

要做到这一点，请遵循以下步骤：

  1. 在 src/ 中进行修改。这个文件夹包含了本网站中使用的所有 SCSS 和 Javascript。你应该在这里编辑这些文件，它们将在网站建立时被包含在 sphinx_book_theme/ 文件夹中。

  2. 安装 tox。

    $ pip install tox

  3. 建立文档。你可以使用下面的 tox 命令：

    $ tox -e docs-update

这将使用最新版本的主题（包括你对 src/ 文件夹所做的任何修改）构建文档，并将输出放在 docs/_build/html 中。然后你可以通过打开其中一个 HTML 文件来预览它们。

当你进行修改时，自动构建文档¶

你也可以使用一个实时服务器来观察主题的文件夹，并自动构建/更新本地文档，以便你可以快速预览变化。这需要使用 sphinx-autobuild 软件包。要做到这一点，使用这个 tox 命令：

$ tox -e docs-live

这将做到以下几点：

  • 生成主题的文档（类似于运行 tox -e docs-update）

  • 在一个可用的端口 localhost:xxxx上启动一个开发服务器

  • 在你的默认浏览器中打开它

  • 观察变化并自动重新生成文档，在发生变化时自动重新加载浏览器。

有了这个，你可以在编辑器中修改主题，并看到这些修改在浏览器上是如何呈现的。

使用不同的 Sphinx 构建器¶

在本地构建文档时，你可以通过向 tox 提供参数来使用不同的 Sphinx 构建器。比如说：

$ tox -e docs-update -- singlehtml
$ tox -e docs-live -- singlehtml

运行测试¶

这个主题有一个测试套件，以确保所有相关的用户内容被正确处理。测试可以通过以下方式运行：

$ tox

你也可以针对不同的 Python 和 Sphinx 版本运行，并向 pytest 提供参数：

$ tox -e py38-sphinx2 -- -x

小技巧

要 “重建” 一个环境，并更新依赖版本，请使用：

$ tox -r -e py38-sphinx2
